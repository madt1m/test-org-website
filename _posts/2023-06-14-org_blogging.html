Incipit

I’ve very recently decided to start blogging – I feel like gathering knowledge is more fun and useful when I find the time to order thoughts and write things down somewhere. And if this can be useful for someone else, all the better.

However, I’m not (yet?) an adept of the dark arts of blogging, and I needed something fast and fully integrated with my workflow to convince myself. Therefore, after a short test with some of the popular WYSIWYG editors online, I decided to integrate my lately most beloved software, Emacs, with my blogging intentions.

And to headstart this blog, what better way to briefly explain the setup of the blog itself?

Enter org-mode to HTML to jekyll to Github Pages.

Ingredients

Org mode

For starters, I need to write Org documents. Emacs (https://www.gnu.org/software/emacs/) is my shop. I am using an Emacs distribution (https://github.com/doomemacs/doomemacs), which sets up really nice defaults and provides a sensible and easy to hack configuration. For all the purist out there…please bear with me but I can’t stand purity: I use Emacs with vim keybindings, and right now Emacs is running within WSL2 (that’s a story for another post) :D

For readers not used to it, org-mode is a “highly flexible plain text file format” that can be used to manage notes, journals, agendas, TODOs, full-fledged calendars, literal programming (jupyter notebooks, I’m sorry about that) and (probably) much more. I highly suggest you to check it out and play with it.

Feel free to check the doc (https://orgmode.org/), or, better, just describe-mode inside Emacs.

HTML (via org-publish)

In order to ship our blogposts using Jekyll, I need to translate our Org documents into HTML.

One of the many features of org-mode is being able to export documents in various formats, among which we find - exactly - HTML. Check https://orgmode.org/manual/Publishing.html for more informations.

Jekyll AKA the Generator

I’m letting Jekyll (https://jekyllrb.com/) the burden to produce the website. For this blog, I’m currently using the Beautiful Jekyll (https://beautifuljekyll.com/) template, which to me seems to touch the right point between minimalism and useful features.

Github Pages

Just Github Pages (https://pages.github.com/), a simple and effective way to host a website with built-in versioning and deploy.

The Workflow   ATTACH

I’ve first setup my personal Github Pages repository pointing to the Beautiful Jekyll template. The steps here are quite simple and already exhaustively documented in the repository documentation at https://github.com/daattali/beautiful-jekyll#build-your-website-in-3-steps, so I’m not going to go into any more details about this.

Now we get to the fun part: connecting the org-mode posts to the website.

Beautiful Jekyll (and, more generically, Jekyll) expect posts in the _posts directory. Typically these posts are written in Markdown and later converted by Jekyll, but nothing prevents us to put plain HTML files in there for Jekyll to process and serve in our website.

Therefore, I have created a blog/ directory in my Org project, and wrote the following config lines in config.el: 

        ;; Org-publish nitty-gritties to make it work with Jekyll and Github Pages
        (setq org-publish-project-alist
        '(("blog"
                :base-directory "~/Dropbox/org/blog"
                :base-extension "org"
                :publishing-directory "~/dev/madt1m.github.io/_posts/"
                :recursive t
                :publishing-function org-html-publish-to-html
                :headline-levels 4
                :html-extension "html"
                :body-only t
                )
          (("images"
                :base-directory "~/Dropbox/org/blog/imgs"
                :base-extension "jpg\\|gif\\|png"
                :publishing-directory "~/dev/madt1m.github.io/assets/img"
                :recursive t
                :publishing-function org-publish-attachment
                ))))

The code here is quite simple. I am setting two different org-publish entries, blog and images, respectively for my blogposts and the assets. Each entry is configured with the source and destination directory, the respective function to publish (==convert or just move) the files, and the extension needed to match the files. In the HTML configuration, only the body is exported, because Jekyll will take care of the header.

There are two more pieces needed to support conversion. First, the name of the exported HTML document will be the name of the Org document, therefore the latter needs to be named with the correct format (YYYY-MM-DD-title.org). Second, I need to include a YAML frontmatter on every post for the Jekyll conversion to work fine. So, at the beginning of every org document in the blog directory, I start things with: 

#+OPTIONS: toc:nil num:nil
#+BEGIN_EXPORT html
---
layout: post
title: "This post is actually an Org-mode note"
permalink: /:title/
tags: [emacs, org-mode]
---
#+END_EXPORT

Of course, this snippet was copied right out of this very post.

Now, to export this post, I’ll only need to run the publishing function which will produce the file right inside my website repository. Publishing it to the internet is then just a commit & push away!
